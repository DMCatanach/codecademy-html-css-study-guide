HTML & CSS Quick Guide

Forms and Validation

Tag or Attribute Usage Notes
<form> Defines the form, which is used to collect user input.
<input> The field where users enter information; takes type and id attributes.
type Specifies what kind of input field to display, such as text, radio, checkbox, and so on.
id Specifies the id of an <input> element. Useful for linking parts of the form together, such as an input field and its label.
<label> Displays a label for an input field to the user. Takes a for attribute.
for An attribute of the <label> element, should match the id of the corresponding <input> element to link them together.
name Required of <input> elements in order to store information. The key in the key-value pair that contains the input.
value Can be used to specify default value for an <input>. Also specified in <option> tags to provide the contents of dropdowns and datalists.
<select> Defines a dropdown list, contains options.
<datalist> Creates a dropdown with pre-defined suggestions. Its id attribute matches list attribute of the input element.
<option> Used with select and datalist; defines an option in the dropdown. Takes a value attribute.
<textarea> Creates multiline input area. Size specified with rows and cols attributes.
maxlength Specifies maximum number of characers in a text field.
max Gives the maximum value acceptable in a number or range input.
min Gives the minimum value acceptable in a number or range input.
step Defines the interval for a number or range input.
required Attribute for input fields that must be filled out before submitting the form.
pattern Supplies a regular expression to check input against to validate user input.
placeholder This attribute is useful for providing an example to the user of how to format their input.

Semantic HTML

Tag Usage Notes
<header> Contains introductory information. Can be used at page level, or within sections and articles.
<nav> This tag wraps the set of navigation links on a page.
<section> Defines a section within a page, generally grouping related content together. May contain <article> elements, or may be embedded within an article.
<article> Contains self-contained information or content, such as a news article or blog post. May contain <section> elements, or be included in a section.
<aside> Contains information or content related to surrounding page content, like a sidebar or extra hints or tips.
<figure> Wraps image content and accomanying caption. Uses an <img> tag to define image.
<figcaption> Defines caption for figures and images. Usually included within <figure> tag to keep the caption associated with the corresponding image, graphic, or figure.
<footer> The section at the foot of the page, containing things like contact information, copyright, sitemap links, site policy links, and so on.

CSS Selectors

Selector Example Usage Notes
Element p Selects an HTML element, such as a paragraph. Any HTML element can be selected by its tag.
ID #id Selects by unique id attribute. Used to select one specific element.
Class .class Selects by class attribute and can be combined with an element selector. Classes can be applied to more than one element.
Universal * Selects everything.
Grouping h1, h2, p Elements that share CSS style rules can be grouped together into one rule set.
